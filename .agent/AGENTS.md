# AI Agent Guidelines for AI OCR Learning Journey

このファイルは、本リポジトリで作業するAIエージェント（Antigravity等）向けの開発ガイドラインと指示書です。

## 1. プロジェクト概要
本プロジェクトは、AI OCR（コンピュータビジョン × 自然言語処理）の仕組みを基礎から学び、実務で通用するAIエンジニア兼データサイエンティストとしてのスキルを習得するための学習用リポジトリです。
単なるAPIの呼び出しにとどまらず、前処理、テキスト検出・認識の内部アルゴリズムの理解、精度評価、NLPによる後処理までのパイプライン構築・実験を行います。

## 2. 技術スタック & 開発環境
- **言語**: Python 3.11+
- **主要ライブラリ**:
  - 画像処理: OpenCV
  - OCRエンジン: EasyOCR, PyTesseract
  - ディープラーニング: PyTorch
  - データ処理・分析: Pandas, NumPy
  - 精度評価: Levenshtein (編集距離)

## 3. ディレクトリ構成と役割
エージェントがコードの追加や修正を行う際は、以下のディレクトリ構成に従ってください。

- `01_easy_ocr/`: Phase 1。簡易OCRスクリプトやテスト用画像・出力を格納する。
- `02_detection_recognition/`: Phase 2。CRAFT/DBNet, CRNN/Transformer等のテキスト検出・認識メカニズムの学習・検証コード。
- `03_data_science/`: Phase 3。画像前処理（二値化、ノイズ除去、傾き補正等）や評価指標（CER, WER, 編集距離の自作）に関する実験用Notebookおよびスクリプト。
- `04_post_processing/`: Phase 4。正規表現やLLMを用いた構造化（JSON化）、名寄せ・自動補正ロジックなどの後処理コード。

## 4. エージェントへの指示・制約事項 (Operational Constraints)
- **言語設定**: ユーザーとの会話、およびコードコメント・docstringは原則として**日本語**で行ってください。
- **実装方針**:
  - ライブラリのラッパーを実装するだけでなく、アルゴリズムの理解に繋がるよう、理論的背景を考慮したコードを書いてください（特にPhase 2やPhase 3）。
  - 各種スクリプトやNotebookを作成した際は、コードの意図や動作確認手順を明記してください。
- **ドキュメントの更新**:
  - 実験結果が得られた場合、[README.md](file:///Users/ymto/Documents/git/ai-ocr-learning-journey/README.md) の「精度実験レポート」テーブルを随時更新してください。既存のフォーマットを崩さないように注意してください。
- **コード品質**:
  - PythonのPEP 8に準拠したクリーンなコードを書いてください。
  - 依存ライブラリを追加した場合は、必ず `requirements.txt` を更新するか、ユーザーに報告してください。
- **確認プロセス**:
  - 作成または修正したコードは、可能であれば実行・テストを行って動作を確認してください。

## 5. コマンド・タスク実行
- **依存関係のインストール**:
  `pip install -r requirements.txt`
- **スクリプトの実行（例）**:
  `python 01_easy_ocr/run_ocr.py`
