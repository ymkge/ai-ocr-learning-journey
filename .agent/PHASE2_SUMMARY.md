# Phase 2: テキスト検出と認識の内部メカニズム分解・理解 サマリ

## 1. 実施内容と実績
本フェーズでは、AI OCRの基本パイプラインを構成する「テキスト検出 (Detection)」と「テキスト認識 (Recognition)」のそれぞれの仕組みを分解し、デモスクリプトを通して内部プロセスを可視化・検証しました。

* **ソースコード**: [02_detection_recognition/text_detection_demo.py](file:///Users/ymto/Documents/git/ai-ocr-learning-journey/02_detection_recognition/text_detection_demo.py)
* **実行した処理**:
  1. **テキスト検出のみを実行**: EasyOCRの `reader.detect()` を用いて、全体画像（`sample.png`）から文字位置情報のみを高速に検出しました。
  2. **1行ごとの切り出し (Crop)**: 検出された座標を元に、OpenCVでテキストエリアを4つの部分画像（`cropped_line_1〜4.png`）に分割し、可視化画像 `detected_areas.png` を保存しました。
  3. **認識モデル (CRNN) のテンソルシミュレーション**: PyTorchで簡易的な `MiniCRNN` クラスを実装し、画像テンソルが各レイヤーを流れる過程で形状（Shape）がどのように変化するかを追跡しました。
  4. **CTC Greedy Decode の実装**: 時系列予測データから「重複した文字をマージし、ブランクを除去して文字列を復元する」アルゴリズムをシミュレートしました。

## 2. テンソル形状（Shape）の変遷
リサイズされた画像がテキストに変換されるまでのデータの流れを数学的に確認しました。

| レイヤー / プロセス | 出力テンソル形状 | 役割と説明 |
| :--- | :--- | :--- |
| **入力画像 (前処理後)** | `[1, 1, 32, 260]` | 高さ32にリサイズされたグレースケール画像 (Batch=1, Ch=1, H=32, W=260) |
| **CNN特徴抽出器** | `[1, 512, 1, 32]` | 特徴チャンネルを512に増やし、高さを1に潰し、幅を32に圧縮 |
| **Sequence変換 (LSTM入力)**| `[32, 1, 512]` | 32コマの時系列データとして再配置 (SeqLen=32, Batch=1, FeatureDim=512) |
| **RNN (Bi-LSTM)** | `[32, 1, 512]` | 左から右への文字順の依存関係（文脈）を双方向LSTMで学習 |
| **全結合層 (クラス確率)** | `[32, 1, 12]` | 各コマ(32時点)における12種類の文字クラスの予測確率を出力 |

## 3. 得られた学びと課題
* **「検出」と「認識」の密接な関係**:
  `cropped_line_2.png` (日本語画像) の切り出し結果を確認したところ、文末の「。」の右側がバウンディングボックスの右端でわずかにカットされていることが判明しました。これにより、後段の「認識」AIが「。」を読み落とす原因となったことから、**テキスト検出の精度が最終的なOCR精度に致命的な影響を与える**ことを実体験として理解しました。
* **CTC (Connectionist Temporal Classification) の重要性**:
  画像上で文字の境界線をあらかじめ定義できなくても、文字の重複マージと `[blank]` の除去を行うCTCアルゴリズムをコードで実装・シミュレーションすることで、アライメント不要で綺麗な文章（`"日本語の認識テストです"`）が復元される仕組みを深く理解できました。
